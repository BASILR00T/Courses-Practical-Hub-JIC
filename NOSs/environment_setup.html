Environment Setup in Hyper-V

Before configuring servers, you need to set up the virtual environment. This involves creating the virtual machines and configuring their network settings.

A. Creating Virtual Machines

  1. Open Hyper-V Manager.
  2. Create three new Virtual Machines (e.g., "DC-Server", "File-Server", "Client-PC").
  3. Create new Virtual Switch (Private and connect all VMs with it).
    • On the main page of Hyper-V click on Virtual Switch Manager
    • Choose Private then click Create Virtual Switch
  4. Install Windows Server on "DC-Server" and "File-Server", and Windows 10/11 on "Client-PC".

B. Assigning Static IP Addresses

Static IPs are crucial for servers. Ensure all machines are on the same subnet.

  1. DC-Server:
    • IP Address: 192.168.1.1
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 192.168.1.1 (It will be its own DNS server)
  2. File-Server:
    • IP Address: 192.168.1.10
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 192.168.1.1 (Points to the DC for name resolution)
  3. Client-PC:
    • IP Address: 192.168.1.2
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 192.168.1.1 (Points to the DC)
