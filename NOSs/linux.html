Network Operating Systems (Linux)

1. File and Directory Management Essentials

Common commands used for navigating and managing files and directories.

Command Description
cd Change Directory. e.g., cd /home
ls List directory contents. e.g., ls -l (long listing), ls -a (hidden files).
pwd Print Working Directory. Show current path.
cat Display file content. e.g., cat file.txt
touch Create new file or update timestamp. e.g., touch newfile.txt
vi editor Text editing using the vi editor. e.g., vi filename
mkdir Make Directory. e.g., mkdir NewFolder
rm Remove files. e.g., rm file.txt
rmdir Remove empty directories. e.g., rmdir EmptyFolder
cp Copy files and directories. e.g., cp source dest
mv Move or rename files and directories. e.g., mv oldname newname

2. User and Group Management

Administrative commands used for user and group lifecycle management.

2.1 User Administration

(Commands requiring elevated privileges)

Create User

[root@comp1 ~]# useradd [options] username
[root@comp1 ~]# id username
uid=1001(username) gid=1001(username) groups=1001(username)
Options:
  • -u: Set UID
  • -g: Set Primary group name/GID
  • -o: Override (allow duplicates)
  • -G: Set Secondary group
  • -c: Add Comment
  • -d: Set Home directory
  • -s: Set Shell

Modify User

[root@comp1 ~]# usermod -L username
[root@comp1 ~]# passwd -S username
username LK 2023-01-01 0 99999 7 -1 (Password locked.)
Options:
  • -l: Change login name
  • -L: Lock the account
  • -U: Unlock the account

Delete User

[root@comp1 ~]# userdel [options] username
[root@comp1 ~]# id username
id: username: no such user
Options:
  • -r: Delete home directory and mail spool

Change User Settings

[root@comp1 ~]# chage -l username
Last password change                                    : Jan 01, 2023
Password expires                                        : never
Password inactive                                       : never
Account expires                                         : never
Minimum number of days between password change          : 0
Maximum number of days between password change          : 99999
Number of days of warning before password expires       : 7

2.2 Group Administration

Create Group

[root@comp1 ~]# groupadd [options] groupname
[root@comp1 ~]# tail -1 /etc/group
groupname:x:1001:
Options: -g (GID), -o (Override).

Modify Group

[root@comp1 ~]# groupmod [options] groupname
[root@comp1 ~]# grep groupname /etc/group
groupname:x:1002:
Options: -g (Set GID), -o (Override), -n (Change group name).

Delete Group

[root@comp1 ~]# groupdel groupname

Group Membership Settings

[root@comp1 ~]# gpasswd -a user1 group1
Adding user user1 to group group1
[root@comp1 ~]# grep group1 /etc/group
group1:x:1001:user1
Options:
  • -a: Add a user
  • -d: Delete a user from group
  • -M: Creating multiple members

3. File and Directory Ownership & Permissions

3.1 Ownership Commands

  • chown (change owner): Changes ownership of a file or directory. 
    [root@comp1 ~]# chown newowner filename
[root@comp1 ~]# ls -l filename
-rw-r--r--. 1 newowner root 0 Mar 15 10:00 filename

[root@comp1 ~]# chown -R newowner directory # Recursive
[root@comp1 ~]# ls -ld directory
drwxr-xr-x. 2 newowner root 4096 Mar 15 10:00 directory
  • chgrp (change group): Changes group owner of a file or directory. 
    [root@comp1 ~]# chgrp newgroup filename
[root@comp1 ~]# ls -l filename
-rw-r--r--. 1 root newgroup 0 Mar 15 10:00 filename

3.2 Permission Concepts

Categories: User (owner), Group (group owner), Other (everyone on system).

Regular Permissions: Read (r), Write (w), Execute (x). The inode section storing this is called the Mode.

Permission On File On Directory
r To display contents of a file. To list contents of a directory.
w To create or append a file. To create files & directories.
x To execute a file. To execute a directory.

3.3 Changing Permissions

chmod (change mode): Changes mode (permissions) of files or directories.

Takes two arguments: Criteria used to change permissions, and Filenames to change.

Use the "a" character to refer to all categories (user, group, other) if changes are identical.

Default Permissions (New Files): Given rw-rw-rw- permissions by default.

# Symbolic Method
[root@comp1 ~]# chmod u+x script.sh  # Add execute permission for User
[root@comp1 ~]# ls -l script.sh
-rwxr--r-- 1 root root 0 Mar 15 10:00 script.sh

[root@comp1 ~]# chmod g-w file.txt   # Remove write permission for Group
[root@comp1 ~]# chmod a=r file.txt   # Set read-only for All

# Numeric Method (r=4, w=2, x=1)
[root@comp1 ~]# chmod 755 script.sh
[root@comp1 ~]# ls -l script.sh
-rwxr-xr-x 1 root root 0 Mar 15 10:00 script.sh

3.4 UMASK

  • Purpose: A mask value (022) is used to mask some bits from the default directory permission.
  • Calculation Example: Default directory permission is 777. After masking with 022, result is 755 (777 - 022 = 755).
  • View UMASK Value:
    Command: umask 
    [root@comp1 ~]# umask
0022

    View from file: vi /etc/bashrc

4. System Power Commands

# Shutdown
[root@comp1 ~]# shutdown -P now   # Power off immediately
[root@comp1 ~]# shutdown -r now   # Reboot immediately
[root@comp1 ~]# shutdown -c       # Cancel scheduled shutdown

# Other aliases
[root@comp1 ~]# reboot
[root@comp1 ~]# poweroff
[root@comp1 ~]# halt              # Halts system (does not power off)
