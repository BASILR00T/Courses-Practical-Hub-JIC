Network Operating Systems (Linux)

1. File and Directory Management Essentials

Command Description
cd Change Directory
ls List directory contents (with parameters)
pwd Print Working Directory
cat Display file content
touch Create new file or update timestamp
vi editor Text editing using the vi editor
mkdir Make Directory
rm Remove files
rmdir Remove empty directories
cp Copy files and directories
mv Move or rename files and directories

VI Editor Basics

Simple guide to write, edit, delete, and save using vi.

  • Open File: vi filename
  • Write (Insert): Press i to start typing.
  • Delete: Press Esc then x (delete character) or dd (delete line).
  • Save & Quit: Press Esc then type :wq and hit Enter.
  • Quit without Saving: Press Esc then type :q! and hit Enter.

2. User and Group Management

2.1 User Administration

(Commands requiring elevated privileges)

Create User

[root@comp1 ~]# useradd <option> <username>
Options:
  • -u: Set UID
  • -g: Set Primary group name/GID
  • -o: Override
  • -G: Set Secondary group
  • -c: Add Comment
  • -d: Set Home directory
  • -s: Set Shell
Verification (Confirmation only):

You can verify creation with:

[root@comp1 ~]# id username
uid=1001(username) gid=1001(username) groups=1001(username)

Modify User

[root@comp1 ~]# usermod <options> <username>
Options:
  • -l: Change login name
  • -L: Lock the account
  • -U: Unlock the account

Delete User

[root@comp1 ~]# userdel <options> <username>
Option:
  • -r: Delete home directory and mail spool

Change User Settings (Password Expiration)

[root@comp1 ~]# chage <username>
Verification (Confirmation only):

List password aging information:

[root@comp1 ~]# chage -l username

2.2 Group Administration

Create Group

[root@comp1 ~]# groupadd <options> <groupname>
Options:
  • -g
  • -o
Verification (Confirmation only):

You can verify with:

[root@comp1 ~]# tail -1 /etc/group
groupname:x:1001:

Modify Group

[root@comp1 ~]# groupmod <options> <groupname>
Options:
  • -g: Set GID
  • -o: Override
  • -n: To change the group name
Verification (Confirmation only):

Verify group changes:

[root@comp1 ~]# grep newgroupname /etc/group
newgroupname:x:1001:

Delete Group

[root@comp1 ~]# groupdel <groupname>

Group Membership Settings

[root@comp1 ~]# gpasswd <options> <groupname>
Options:
  • -a: Add a user
  • -d: Delete a user from group
  • -M: Creating multiple members
Verification (Confirmation only):

You can verify membership with:

[root@comp1 ~]# grep groupname /etc/group
groupname:x:1001:user1

3. File and Directory Ownership & Permissions

3.1 Ownership Commands

  • chown (change owner): Changes ownership of a file or directory.
    • Requires two arguments: New owner and File or directory to change.
    • Recursive Option: Can use the -R option to change permissions recursively throughout the directory tree.
  • chgrp (change group): Changes group owner of a file or directory.
    • Uses the same arguments and options as chown.

Change Ownership (chown)

[root@comp1 ~]# chown user1 file1.txt
[root@comp1 ~]# chown user1:root file1.txt # Change owner and group
[root@comp1 ~]# chown -R user1 directory1 # Recursive
Verification (Confirmation only):

Verify ownership changes:

[root@comp1 ~]# ls -l file1.txt
-rw-r--r-- 1 user1 root 0 Mar 15 10:00 file1.txt

Change Group Ownership (chgrp)

[root@comp1 ~]# chgrp group1 file1.txt

3.2 Permission Concepts

  • Permissions Categories: User (owner), Group (group owner), Other (everyone on system).
  • Regular Permissions: Read (r), Write (w), Execute (x).
  • The inode section that stores permissions is called the Mode.

Access Mode Functions:

Permission On File On Directory
r To display contents of a file. To list contents of a directory.
w To create or append a file. To create files & directories.
x To execute a file. To execute a directory.

3.3 Changing Permissions

  • chmod (change mode): Changes mode (permissions) of files or directories.
    • Takes two arguments minimum: Criteria used to change permissions, and Filenames to change.
    • Use the "a" character to refer to all categories (user, group, other) if changes are identical.
  • Default Permissions:
    • New Files: Given rw-rw-rw- permissions by default.

Examples (Symbolic Mode)

[root@comp1 ~]# chmod u+x script.sh
[root@comp1 ~]# chmod u=-rwx,g=-rwx,o=-rwx file1.txt
[root@comp1 ~]# chmod a=rwx file1.txt
Verification (Confirmation only):

You can verify the permissions with:

[root@comp1 ~]# ls -l file1.txt
-rwxrwxrwx 1 root root 0 Mar 15 10:00 file1.txt

Examples (Numeric Mode)

[root@comp1 ~]# chmod 755 script.sh

3.4 UMASK

  • Purpose: A mask value (022) is used to mask some bits from the default directory permission.
  • Default Calculation Example: The default directory permission is 777. After masking with 022, the result is 755 (777 - 022 = 755).
  • View UMASK Value:
    • Command: umask
    • View from file: vi /etc/bashrc

4. System Power Commands

# Shutdown
[root@comp1 ~]# shutdown -P now   # Power off immediately
[root@comp1 ~]# shutdown -r now   # Reboot immediately
[root@comp1 ~]# shutdown -c       # Cancel scheduled shutdown

# Other aliases
[root@comp1 ~]# reboot
[root@comp1 ~]# poweroff
[root@comp1 ~]# halt              # Halts system (does not power off)

5. Data Generation & Redirection (yes command)

The yes command is used to output a string repeatedly until killed. It is often used for automating scripts or generating test data.

Example: Generate 200 lines of "lol"

We use yes to generate the text "lol" repeatedly. We pipe (|) this output to head -n 200, which takes only the first 200 lines. Finally, we redirect (>) the result to a file named test.txt.

[root@comp1 ~]# yes lol | head -n 200 > test.txt
                
[root@comp1 ~]# cat test.txt
lol
lol
... (200 times)
