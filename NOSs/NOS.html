Network & Server Configuration Guide

1. Environment Setup in Hyper-V

Before configuring servers, you need to set up the virtual environment. This involves creating the virtual machines and configuring their network settings.

A. Creating Virtual Machines

  1. Open Hyper-V Manager.
  2. Create three new Virtual Machines (e.g., "DC-Server", "File-Server", "Client-PC").
  3. Install Windows Server on "DC-Server" and "File-Server", and Windows 10/11 on "Client-PC".

B. Assigning Static IP Addresses

Static IPs are crucial for servers. Ensure all machines are on the same subnet.

  1. DC-Server:
    • IP Address: 192.168.1.10
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 127.0.0.1 (It will be its own DNS server)
  2. File-Server:
    • IP Address: 192.168.1.11
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 192.168.1.10 (Points to the DC for name resolution)
  3. Client-PC:
    • IP Address: 192.168.1.100
    • Subnet Mask: 255.255.255.0
    • Default Gateway: (Your network's gateway, e.g., 192.168.1.1)
    • Preferred DNS: 192.168.1.10 (Points to the DC)

2. Create a Domain Controller (DC)

The Domain Controller is the heart of a Windows network, managing security and user authentication.

A. Add the AD DS Role

  1. On your "DC-Server" VM, open Server Manager.
  2. Click "Add roles and features".
  3. Choose "Role-based or feature-based installation".
  4. Select your server from the server pool.
  5. Check the box for Active Directory Domain Services. Add the required features when prompted.
  6. Complete the wizard by clicking "Next" through the remaining steps and finally "Install".

B. Promote Server to a Domain Controller

  1. After the installation, a notification flag will appear in Server Manager. Click it and select "Promote this server to a domain controller".
  2. Select "Add a new forest".
  3. Enter a Root domain name (e.g., `jic.local`).
  4. Set a DSRM (Directory Services Restore Mode) password. This is a crucial recovery password.
  5. Proceed through the wizard, accepting defaults, and finally click "Install". The server will restart automatically.

3. Create a File Server

A file server provides centralized storage and access to files on the network.

A. Add the File Server Role

  1. On your "File-Server" VM, open Server Manager.
  2. Go to "Add roles and features".
  3. Under Server Roles, expand "File and Storage Services" -> "File and iSCSI Services", and check File Server.
  4. Complete the wizard to install the role.

B. Create a Shared Folder

  1. In Server Manager, go to "File and Storage Services" -> "Shares".
  2. In the "SHARES" tile, click "TASKS" -> "New Share...".
  3. Choose "SMB Share - Quick" and follow the wizard to specify a share location (e.g., C:\SharedDocs) and name.
  4. Configure permissions as needed (e.g., give "Domain Users" read/write access).

4. Configure Disk Volumes (RAID)

Combine multiple physical disks into a single logical volume for performance or redundancy. This requires adding extra virtual hard disks to your server VM in Hyper-V settings.

  1. Open Disk Management on your server.
  2. Initialize the new disks.
  3. Right-click on an unallocated disk space and choose the desired volume type:
    • Spanned Volume: Combines space from multiple disks into one large volume. No redundancy.
    • Striped Volume (RAID 0): Writes data across multiple disks for faster performance. No redundancy.
    • Mirrored Volume (RAID 1): Duplicates data onto two separate disks for excellent redundancy.
  4. Follow the wizard to select disks and complete the creation.

5. Manage OUs, Users, and Groups

Organize your network resources and manage user access through Active Directory.

  1. On your "DC-Server", open Active Directory Users and Computers from the Tools menu in Server Manager.
  2. Create an OU (Organizational Unit): Right-click your domain -> New -> Organizational Unit. (e.g., "Students", "Faculty").
  3. Create a Group: Right-click the new OU -> New -> Group. (e.g., "COIT-Students").
  4. Create a User: Right-click the OU -> New -> User. Fill in the details and set a password.
  5. Add User to Group: Right-click the new user -> "Add to a group...". Enter the group name ("COIT-Students") and click OK.

6. Join a Client PC to the Domain

Connect your client machine to the domain to enable centralized login and management.

  1. On your "Client-PC" VM, right-click the Start Menu and go to System.
  2. Click on "Rename this PC (advanced)" or "Advanced system settings".
  3. Go to the "Computer Name" tab and click "Change...".
  4. Under "Member of", select Domain and type your domain name (e.g., `jic.local`).
  5. Click OK. You will be prompted for credentials. Enter the username and password of a domain administrator account.
  6. You should see a welcome message. Restart the PC to apply the changes. You can now log in with the domain user account you created.
