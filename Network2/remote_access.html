Remote Access (Telnet & SSH)

Telnet (Insecure)

Telnet provides basic, unencrypted remote access. It should only be used in secure, isolated lab environments. The configuration involves setting a password on the VTY (Virtual Teletype) lines.

! Configure the virtual terminal lines for remote access
Switch(config)# line vty 0 15
! Set a password for Telnet access
Switch(config-line)# password cisco
! Enforce login with the password
Switch(config-line)# login

SSH (Secure Shell)

SSH provides secure, encrypted remote access and is the industry standard. Configuration is more involved as it requires creating a local user, defining a domain name, and generating encryption keys.

  1. Set a hostname and IP domain name.
  2. Generate crypto keys.
  3. Create a local username and password for authentication.
  4. Configure VTY lines to use SSH and authenticate against the local user database.
Router(config)# hostname MyRouter
MyRouter(config)# ip domain-name mylab.local
MyRouter(config)# crypto key generate rsa
The name for the keys will be: MyRouter.mylab.local
Choose the size of the key modulus in the range of 360 to 2048 for your
  General Purpose Keys. Choosing a key modulus greater than 512 may take
  a few minutes.

How many bits in the modulus [512]: 1024
% Generating 1024 bit RSA keys, keys will be non-exportable...[OK]

MyRouter(config)# username admin secret cisco123
MyRouter(config)# line vty 0 4
MyRouter(config-line)# transport input ssh
MyRouter(config-line)# login local
