Network Services (DHCP & DNS)

DHCP Server on a Cisco Router

A Cisco router can be configured to act as a DHCP server, automatically assigning IP addresses to clients on the network.

! Exclude specific IP addresses from being assigned (e.g., for servers, router itself)
Router(config)# ip dhcp excluded-address 192.168.1.1 192.168.1.10

! Create a DHCP pool with a specific name
Router(config)# ip dhcp pool LAN-POOL-1
! Define the network and subnet mask for the pool
Router(dhcp-config)# network 192.168.1.0 255.255.255.0
! Specify the default gateway for clients
Router(dhcp-config)# default-router 192.168.1.1
! Specify the DNS server for clients
Router(dhcp-config)# dns-server 8.8.8.8

DNS Configuration on a Client

Clients need to be configured with the IP address of a DNS server to resolve domain names (like www.google.com) into IP addresses. This is typically assigned via DHCP, but can be set manually.

Verification on a Cisco Router (acting as client)

! Set the DNS server for the router itself
Router(config)# ip name-server 8.8.8.8

! Test DNS resolution
Router# ping www.cisco.com
