VLSM & IP Addressing

Understanding VLSM

Variable Length Subnet Masking (VLSM) is a technique that allows network administrators to divide an IP address space into subnets of different sizes. This is far more efficient than traditional "classful" subnetting, as it minimizes wasted IP addresses.

For example, you can use a /30 mask (2 host IPs) for a point-to-point link between routers and a /27 mask (30 host IPs) for a user LAN, all from the same original network block.

Calculating Subnets

To use VLSM, you need to understand how to calculate the Network ID, Broadcast Address, and the range of usable host IPs for any given IP address and subnet mask.

Subnet Mask Quick Reference

CIDR Subnet Mask Total IPs Usable Hosts
/24 255.255.255.0 256 254
/25 255.255.255.128 128 126
/26 255.255.255.192 64 62
/27 255.255.255.224 32 30
/30 255.255.255.252 4 2

Example Calculation

Given the IP address 192.168.200.139 with a mask of 255.255.255.224 (/27):

  • Network Address: 192.168.200.128
  • Broadcast Address: 192.168.200.159
  • First Usable Host IP: 192.168.200.129
  • Last Usable Host IP: 192.168.200.158

Practical Example: Assigning IPs in Packet Tracer

Assigning an IP address to a PC in Packet Tracer is a fundamental skill for testing connectivity in any lab.

  1. Click on the PC in the Packet Tracer workspace.
  2. Go to the **Desktop** tab.
  3. Click on **IP Configuration**.
  4. Fill in the static IPv4 Address, Subnet Mask, and Default Gateway. The Default Gateway is the IP address of the router interface on the same network, which the PC will use to reach other networks.
