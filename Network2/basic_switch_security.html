Basic Switch Security

Securing Switch Access

Securing access to the switch's command-line interface (CLI) is the first and most critical step in network security. This involves setting passwords for different access levels to prevent unauthorized configuration changes.

1. Console Port Security

The console port provides direct physical access to the switch. Securing it prevents anyone with a console cable from gaining control.

Using a Simple Password

This method sets a single password for console access.

! Enter global configuration mode
Switch>enable
Switch#configure terminal
! Enter console line configuration mode
Switch(config)#line console 0
! Set the password and require it for login
Switch(config-line)#password cisco
Switch(config-line)#login

Using a Local Username and Password

This is a more secure method as it requires both a username and a password, allowing for better user tracking.

! First, create a local user in the database
Switch(config)#username admin password cisco123
! Then, configure the console to use this local database
Switch(config)#line console 0
Switch(config-line)#login local

2. Enable Mode (Privileged EXEC) Security

Securing the enable mode is crucial as it grants access to all configuration commands.

Enable Password (Less Secure)

The enable password is stored in plain text in the configuration by default, making it less secure.

Switch(config)#enable password class

Enable Secret (More Secure)

The enable secret command is the **preferred method**. It uses a strong MD5 hash to encrypt the password, making it much more secure. It always overrides the `enable password` if both are configured.

Switch(config)#enable secret class

3. Password Encryption

By default, passwords like the console and Telnet passwords are stored in plain text. The service password-encryption command encrypts these passwords in the configuration file, adding a basic layer of security.

Note: This uses a weak encryption algorithm and is not a substitute for using `enable secret`, but it's good practice to enable it.

! Enable encryption for all plain-text passwords
Switch(config)#service password-encryption

! Viewing the running config will now show encrypted passwords
Switch#show running-config
...
enable password 7 0822455D0A16
...
line console 0
 password 7 0822455D0A16
...
