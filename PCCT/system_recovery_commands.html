System Recovery & Backup

These commands are typically run from the Windows Recovery Environment (WinRE) Command Prompt to troubleshoot and repair boot and system file issues.

Accessing Windows RE

If Windows can still boot, you can force it to restart into the Recovery Environment with the following command:

C:\>shutdown /r /o
[A message appears that you're about to be signed out]

1. System Protection / Create a Restore Point

To use System Restore, System Protection must be enabled on your drive (usually C:). This allows Windows to automatically or manually create restore points.

A. System Protection Tab

This setting is found in the System Properties dialog box. You can access it by searching for "Create a restore point" in the Start menu.

B. Turn on System Protection

Click the Configure button and select "Turn on system protection" to enable it for the selected drive.

C. Create a Manual Restore Point

Once enabled, you can click the Create button and provide a name/description to save the current system state.

Note: System Restore only affects system files, installed programs, and registry settings. It does not backup or affect your personal files (documents, photos, etc.).

2. System Restore

If your system becomes unstable, you can revert to a previous state using the System Restore wizard.

rstrui.exe

This is the executable command that launches the System Restore GUI directly.

rstrui.exe
[The System Restore window opens, allowing you to choose a restore point]

Other Backup Tools

Backup and Restore (Windows 7)

Windows 10 includes the legacy Backup and Restore (Windows 7) tool, found in the Control Panel. It allows for creating full system images and file backups.
