System Properties and startup

System Properties (sysdm.cpl)

The System Properties dialog is a central hub for many advanced settings. It can be accessed quickly by running the command sysdm.cpl.

To open System Properties: Press Windows Logo + R, type sysdm.cpl, and press Enter.

Managing Virtual Memory

Virtual memory (the paging file) is a space on your hard disk that Windows uses as if it were RAM. You can manage its size and location for performance tuning.

  1. Open System Properties (sysdm.cpl).
  2. Go to the Advanced tab.
  3. Under the "Performance" section, click Settings....
  4. In the Performance Options window, go to the Advanced tab.
  5. Under the "Virtual memory" section, click Change....
  6. From here you can uncheck "Automatically manage..." to set a custom size or move the pagefile.sys to a different drive.

Configuring Startup Applications

I. Using Startup Folders (GUI Method)

This method involves placing a shortcut to the desired application inside a designated Startup folder.

User Scope: All Users

Folder Path: C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup

Key Action:

  1. Create a Shortcut of the application.
  2. Put the Shortcut in the Startup Folder listed above.

User Scope: Current User / Specific User

Folder Path: %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup
(e.g., C:\Users\Student-Admin\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup)

Key Action: Add the Shortcut to this folder.

Note: For "Specific User", you must have Admin privileges to access their folder.

II. Using the Windows Registry (Advanced Method)

This method involves adding a registry string value that points to the application's executable file.

User Scope: All Users

Registry Key: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

Key Action:

  1. Copy the full file path of the application (e.g., C:\Windows\System32\cmd.exe).
  2. Navigate to the registry key above.
  3. Add a new String Value.
  4. Paste the application path as the value data.

User Scope: Current User

Registry Key: HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

Key Action: Follow the same steps as for "All Users", but in the Current User hive.

Important Notes:

  • WARNING: Always back up the registry key by exporting it before making changes.
  • Ensure the App name is included in the Path/Value name for clarity.
  • Verify success by checking Task Manager > Startup to see if the icon appears.

Microsoft Management Console (MMC)

5.1 What is the MMC?

  • Microsoft Management Console (MMC): An application that simplifies many administrative tasks by allowing the collection of many tools (called snap-ins) inside a single console.
  • Function: It is an application framework that hosts administrative tools ("snap-ins") and allows administrators to create a single, custom, and portable interface.

5.2 Key Benefits of the MMC

  • Tool Consolidation: You can collect numerous snap-ins (like Event Viewer, Disk Management, Device Manager) into a single console, improving efficiency.
  • Remote Management: Many snap-ins can be configured to connect to and manage remote computers, enhancing the administrator's reach across the network.

5.3 Guide to Creating a Custom Console

  1. Run the Microsoft Management Console (MMC) by executing mmc.exe.
  2. Click File > Add/Remove Snap-in... from the menu bar.
  3. In the dialog, select a desired tool from the list of available snap-ins and click Add >.
  4. Repeat for all desired tools.
  5. Click OK to add the snap-ins to the console.
  6. To save the custom console, click File > Save As... and provide a descriptive name and location.

Note: Saved console files have the .msc extension. You can also set console modes (Author/User) under File > Options to control what users can do with the console.
